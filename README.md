
# MeetingQuality Platform

Система мониторинга качества совещаний с поэтапным процессом оценки.

## Основной функционал
1. **Регистрация и Вход**: Простое управление пользователями.
2. **Дашборд**: Управление встречами и задачами с фильтрацией (Текущие/Прошедшие).
3. **Трехэтапные встречи**:
   - Фаза 1: Обсуждение основного вопроса.
   - Фаза 2: Оценка понимания и эмоционального вклада (включая токсичность).
   - Фаза 3: Подведение итогов (постановка персональных задач).
4. **Статистика**: Анализ результатов завершенных встреч.
5. **Real-time**: Синхронизация состояний фаз между участниками через имитацию WebSocket (polling).

## Технологии
- **Frontend**: React 18, TypeScript, Tailwind CSS.
- **State Management**: React Context + LocalStorage для имитации бэкенда.
- **Backend (Proposed)**: NestJS (текущая версия имитирует его логику через `store.tsx`).

## Запуск проекта
1. Установите зависимости (если используется локально): `npm install`.
2. Запустите проект: `npm start`.
3. Приложение будет доступно по адресу `http://localhost:3000`.

## Инструкция для Бэкенда (NestJS)
Для реализации полноценного бэкенда рекомендуется:
1. Создать модули `User`, `Meeting`, `Task`.
2. Использовать `Socket.io` для реализации мгновенного переключения фаз.
3. Хранить данные в PostgreSQL/MongoDB.
4. Добавить JWT авторизацию для безопасности.
